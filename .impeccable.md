## Design Context

### Users
Developers and technical evaluators use the iVALT Developer Portal to access biometric authentication API documentation, manage API keys, and understand the integration flow quickly. They need confidence that the system is secure, stable, and ready for production use.

### Brand Personality
Premium, secure, calm. The interface should feel trustworthy and controlled rather than playful or noisy.

### Aesthetic Direction
Light-mode-first premium security console with refined surfaces, warm off-white canvas, iVALT purple as the primary brand anchor, restrained blue accents for API/documentation actions, and clear operational status cues.

### Design Principles
Lead with trust and clarity before decoration.
Make key capacity and next actions immediately legible.
Use restrained contrast, generous whitespace, and precise typography to convey security.
Keep developer workflows fast: API keys, docs, and auth flow should always be one obvious click away.
